4 A Note on Namespaces
5 --------------------
6 
7 All interfaces live within the `Farsyte` namespace, to avoid
8 collisions with simiarly named things defined elsewhere.

10 Header File Overview
11 ====================
12 
13 Farsyte::Utility "utility.hh"
14 ----------------
15 
16 The `utility.hh` header collects general utility code that
17 is not associated with any of the larger packages within
18 the toolkit project. This could be code that is shared
19 across multiple packages (without an obvious owner), or it
20 could be entire packages that are simply too small to
21 warrant tracking as their own library.
22 
23 Farsyte::Testing "testing.hh"
24 ----------------
25 
26 The `testing.hh` header provides declarations for the API
27 for the library that is used to record test results in a
28 form that allows an automated build system (currently the
29 Bamboo continuous intergration system from Atlassian) to
30 track testing results.
31 
32 The `Testing` Library implements four classes:
33 
34  * The [Oops](classFarsyte_1_1Testing_1_1Oops.html) class
35  is the base class for exceptions thrown within
36  the `Testing` library representing failures of
37  the library or failure to follow the requirements
38  of the library when calling it.
39 
40  * The [Test](classFarsyte_1_1Testing_1_1Test.html) class
41  represents a single test, counting as one test when presenting
42  counts of tests passed or failed. Tests may indicate one or more
43  conditions that fail or were skipped, with supporting text, as
44  well as errors occurring in the testing process.
45  It is not uncommon for a Test to correspond to a single method
46  name within a class, and for the code to test various aspects of
47  the method in sequence.
48  If two Test objects have the same test name and the same suite
49  name, their results will be combined during reporting.
50 
51  * The [Suite](classFarsyte_1_1Testing_1_1Suite.html) class
52  represents a collection of Test objects that are logically
53  related; tests with the same Suite name are grouped together when
54  reporting summaries of test results.
55  It is not uncommon for a Suite to correspond to a single Class to
56  be tested within a library.
57  If two Suite objects have the same name, their results will be
58  combined during reporting (even if they come from tests in quite
59  different subprojects). Risks associated with this can be
60  mitigated by assuring that class names are distinct between
61  subprojects of a superproject, when tests are run across the
62  entire superproject as a unit.
63 
64  * The [Log](classFarsyte_1_1Testing_1_1Log.html) class
65  corresponds to a single stream of test output produced by
66  sequentially running a series test suites, each of which
67  sequentially runs a series of test cases. Note that the Log class
68  allows test programs to have multiple Log objects open at once,
69  allowing a test program to persue multiple test series in parallel
70  if this is appropriate and useful.
71 
72  * The [Fixture](classFarsyte_1_1Testing_1_1Fixture.html) class
73  provides a simple light testing framework that encapsulates
74  the use of the various Testing classes above within macros
75  intended to make the tests themselves easier to read. Code
76  using this facility should use it via the varios `UT_` macros
77  provided in the header files.

79 Farsyte::Simple "simple.hh"
80 ---------------
81 
82 The `simple.hh` header provides declarations for the API
83 for the library that implements non-template versions of
84 Matrix, Vector, and Quaternion math. The simplification
85 is that the components are all `double` and the dimensions
86 are all `THREE` (except of course Quaternion, which is
87 a scalar plus a three-vector).
88 
89  * The [Col](classFarsyte_1_1Simple_1_1Col.html) class
90  encapsulates a column three-vector. Operations
91  supported include (C Struct style) construction,
92  comparison, subscripting, Add/Subtract, Scale
93  up and down by a scalar value, dot product, cross
94  product, square of the two-norm and the two-norm,
95  as well as a simple "print to stream" method.
96  This class is the preferred representation for
97  use when manipulating three-vectors.
98 
99  * The [Row](classFarsyte_1_1Simple_1_1Row.html) class
100  encapsulates a row three-vector. Operations
101  supported include (C Struct style) construction,
102  comparison, subscripting, Add/Subtract, Scale
103  up and down by a scalar value, dot product, cross
104  product, square of the two-norm and the two-norm,
105  as well as a simple "print to stream" method.
106 
107  * The [Mat](classFarsyte_1_1Simple_1_1Mat.html) class
108  encapsulates a three-by-thee matrix. Operations
109  supported include a number of constructors,
110  comparison, subscripting, Add/Subtract, Scale
111  up and down by a scalar value, matrix multiply,
112  multiply matrix by column vector, and a simple
113  "print-to-stream" method.
114 
115  * The [Quat](classFarsyte_1_1Simple_1_1Quat.html) class
116  encapsulates a Quaternion (four coefficient
117  hypercomplex variable). Operations
118  supported include a number of constructors,
119  comparison, component access, Add/Subtract, Scale
120  up and down by a scalar value, square of the
121  two-norm, two-norm, quaternion multiply,
122  inversion, quaternion divide, extraction of a
123  corresponding scale-and-rotate matrix, operating
124  directly on a vector, and a simple
125  "print-to-stream" method.

127 Farsyte::Matrix "matrix.hh"
128 ---------------
129 
130 The template-based Matrix / Vector / Quaternion code is
131 currently on hold pending updates to development tools.
132 
133 Matrix math is provided as a generalized template for dense
134 rectangular matrices, a derived template for working with column
135 vectors, and a class representing a position in a three dimensional
136 state space (such as distance north, east, and above the flagpole).
137 
138 The `Matrix` Library implements two templates and one class:
139 
140  * The [Matrix](classFarsyte_1_1Matrix_1_1Matrix.html) class
141  template supports dense rectangular matrices with any small
142  positive integer number of rows and columns, containing data
143  elements of any data type that supports the desired operations.
144  This class currently provides class methods retrieving the
145  dimensions of the matrix, and member methods providing the
146  operations listed here. More operations will be added once I am
147  happy with code quality, test coverage, and documentation for
148  these operations:
149  * Initialization
150  * Addition
151  * Difference
152  * Negation
153  * Transpose
154 
155  * The [ColVec](classFarsyte_1_1Matrix_1_1Colvec.html) class template
156  is a proxy class providing the obvious API adjustments to use
157  for column vectors, which are simply a one-column matrix.
158  The API differences mainly involve not having to redundantly
159  specify column numbers when working with a ColVec class.
160 
161  * The [TriVec](classFarsyte_1_1Matrix_1_1TriVec.html) class
162  likewise provides for a three-element column vector such as might
163  be used to represent a Position (north, east and above the
164  flagpole), Velocity, Direction, and so on.
165  The `TriVec` class adds the Cross Product operation, which is
166  not provided for general Matrix and Column Vector objects.
